Crumb LLM

A new kind of language model. Physics-based. O(N log N). Structure-aware.

What if attention was a wave?

Traditional transformers compare every token to every other token — O(N²) cost, O(N²) memory. Double the context, quadruple the price. Crumb LLM replaces all of that with a wave equation. Each token deposits information onto a continuous field; a damped-cosine kernel propagates it via FFT in O(N log N); tokens read back from the field at their positions. No quadratic attention matrix. No KV-cache that grows with history. Just physics.

The cost difference

Where a transformer's memory grows with context length, Crumb LLM's field-cache is fixed.

Property Traditional Transformer Crumb LLM
Mixing Every token × every token Scatter → FFT → Gather
Per-layer cost O(N²) O(F log F)
Cache KV-cache grows with N Field-cache is fixed
32K context ~35 GB 6.7 GB

Three physics parameters per head

A standard transformer head spends 2d² parameters on Q/K projections alone — 32,768 at d=128. Crumb LLM V1 routes information with three learnable physics values.

Parameter Symbol What it controls
Damping α How far information travels. High α = local head, low α = global head.
Frequency ω Which spatial patterns the head is sensitive to. Like tuning a radio.
Phase φ Where in the oscillation cycle the kernel starts. Shifts the head's viewpoint.

The kernel:

k(t) = exp(−α|t|) · cos(ωt + φ)

V2 blocks add learned components (smooth causal masking, learned scatter/gather, query-conditioned frequency gating, RoPE, short convolution) and remain O(N log N). V2 is the SOTA variant; V1 is the minimal, fast one.

1Wave-field mixing exploits structure

CRUMB documents have repetitive structure (section headers, keywords, patterns) that maps naturally onto spectral components. The FFT kernel captures these in one O(F log F) operation where attention needs O(N²) pairwise comparisons to notice the same regularities.

2Context pulling via spectral matching

The spectral fingerprint of a section acts like an embedding — computed from the same physics as the model itself, not a separate encoder. Querying 110 indexed sections returns the right one in under 2ms. The model and the retriever speak the same language.

3O(1) generation via field-state cache

Because convolution is linear, the convolved field is cached and each new token only adds its own contribution. Cost per generated token is O(F log F), independent of history length. A transformer's KV-cache costs O(N) per token and grows without bound.

4Information decays, it doesn't cliff-drop

In sliding-window attention, a token that scrolls past the window is gone. In Crumb LLM's sliding field, old tokens leave residual wave energy that decays gracefully by the damping coefficient α — a token 10,000 positions back still has non-zero influence if its α was small enough.
